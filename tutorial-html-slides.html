100vh 规则
不允许页内滚动 clamp 响应式

为什么要严格限制

Why
  • 1
    舞台感每次滚轮/按键=翻页，不是“往下拉一点”
  • 2
    兼容性投屏/移动端/浏览器缩放都能完整显示
  • 3
    逼迫做减法文字过多就拆页，信息更清晰
  • 4
    性能更稳可控的 DOM 数量与图片尺寸

最小检查清单

Checklist
  • ✓
    .slide 100vhheight: 100vh/100dvh + overflow: hidden
  • ✓
    字体间距 clamp不要用固定 px/rem 作为主单位
  • ✓
    图片限制max-height: min(50vh, 400px)
  • ✓
    短屏降级600px 以下隐藏非关键装饰
拆页优先
少即是多

核心 JS 结构
IntersectionObserver Keyboard/Touch

为什么用 Observer

Motion
  • 1
    性能稳定不用每帧监听滚动做计算
  • 2
    动效可控进入视口后一次性触发
  • 3
    可降级减少动画的用户自动忽略
  • 4
    配合 snap一页一页的节奏更清晰

关键片段（8 行）

Code
const io = new IntersectionObserver( (entries) => entries.forEach((e) => e.isIntersecting && e.target.classList.add("visible")), { threshold: 0.35 } ); document.querySelectorAll(".slide").forEach((s) => io.observe(s));

配合 CSS：.slide.visible .reveal { opacity: 1; transform: none; }

常见坑
溢出/空白 字体/图片

最容易踩的 6 个问题

Troubleshoot

内容溢出

一页放太多要点或图，拆页解决。

空白过多

用网格/卡片填满，目标≥85%视觉覆盖。

字体太小

用 clamp 统一缩放，不手写固定 px。

图片过大

限制高度 + 压缩大小，避免加载慢。

动效过猛

只在关键点用，避免满屏飞。

移动端抖动

短屏隐藏装饰，减少阴影层数。
